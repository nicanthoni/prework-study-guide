Javascript

  • JavaScript is case sensitive. This means 'myVariable' is not the same as 'myvariable'.
  • // This is a comment (in Javascript & CSS). This can be placed on its own line, or right after some code.
  • A multi-line comment will comment out multiple lines and is denoted with /* to begin the comment, and */ to end the comment.
  • Data types are the classifications we give to the different kinds of data that we use in programming. In JavaScript, there are seven fundamental data types: Number, String, Boolean, Null, Undefined, Symbol, & Object.
  • For types, Conceptually, undefined indicates the absence of a value, while null indicates the absence of an object (which could also make up an excuse for typeof null === "object").
  • An operator (+, -, *, /, and %) is a character that performs a task in our code. JavaScript has several built-in arithmetic operators, that allow us to perform mathematical calculations on numbers.
  • Operators aren’t just for numbers! When a + operator is used on two strings, it appends the right string to the left string
  • console.log('no' + 'space'); // Prints 'nospace'
  • console.log('Hello'.length); // Prints 5. The ' . ' is another operator! We call it the dot operator. In the example here, the value saved to the length property is retrieved from the instance of the string, 'Hello'. The program prints 5 to the console, because Hello has five characters in it.
  • Methods are actions we can perform. Data types have access to specific methods that allow us to handle instances of that data type. JavaScript provides a number of string methods.
  • We call, or use, these methods by appending an instance with: a period(the dot operator),the name of the method, opening and closing parenthesis. E.g. 'example string'.methodName(). When we use console.log() we’re calling the .log() method on the console object.
  • a variable is simply a named container that allows us to store data in our code.
  • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Math.floor() and Math.random() are both properties of JavaScript's built-in Math object that can be used together to generate a random number.
  • Math.ceil() returns the smallest integer (whole number) greater than or equal to 'x'.
  • Math.random() generates a random number between 0 and 1. Math.floor() rounds the number down to the nearest whole number.
  • Number.isInteger() determines whether the passed value is an integer.
  • We can insert, or interpolate, variables into strings using template literals.
  • A template literal is wrapped by backticks ` (this key is usually located on the top of your keyboard, left of the 1 key). Inside the template literal, you’ll see a placeholder, ${myVar}. The value of myVar is inserted into the template literal. When we interpolate `I own a pet ${myPet}.`, the output we print is the string: 'I own a pet cat.'
  • console.log(`My name is ${myName}. My favorite city is ${myCity}`) = My name is Nic. My favorite city is London.
  • typeof operator: checks the value to its right and returns, or passes back, a string of the data type.
  • let myName = 'Nic'.toLowerCase() creates a myName string variable that will always display in lowercase: "nic"
  • A conditional statement checks a specific condition(s) and performs a task based on the condition(s). Think if, else if, and else statements
  • When writing conditional statements, sometimes we need to use different types of operators to compare values. These operators are called comparison operators.
  • Think of comparison statements as questions. When the answer is “yes”, the statement evaluates to true, and when the answer is “no”, the statement evaluates to false.
  • We can use a ternary operator to simplify an if...else statement. Example: isNightTime ? console.log('Turn on the lights!') : console.log('Turn off the lights!');
  • The else if statement allows for more than two possible outcomes. if/else if/else statements are read from top to bottom, so the first condition that evaluates to true from the top to bottom is the block that gets executed.
  • A switch statement provides an alternative syntax that is easier to read and write, versus needing to write 100 else if statements to check multiple values.
  • - The switch keyword initiates the statement and is followed by ( ... ), which contains the value that each case will compare.
  • - Inside the block, { ... }, there are multiple cases. The case keyword checks if the expression matches the specified value that comes after it. The value following the first case is 'tomato'. If the value of groceryItem equalled 'tomato', that case‘s console.log() would run.
  • - The break keyword tells the computer to exit the block and not execute any more code or check any other cases inside the code block.
  • - At the end of each switch statement, there is a default statement. If none of the cases are true, then the code in the default statement will run.
  • A function declaration consists of: (1) The function keyword, (2) The name of the function, or its identifier, followed by parentheses, & (3) A function body, or the block of statements required to perform a specific task, enclosed in the function’s curly brackets, { }.
  • A function declaration binds a function to an identifier. The code inside a function body runs, or executes, only when the function is called.
  • To call a function in your code, you type the function name followed by parentheses. Ex: calling "sayThanks();" would run wahatever I declared when first coding function sayThanks() {};A function call executes the function body, or all of the statements between the curly braces in the function declaration.
  • When declaring a function, we can specify its parameters. Parameters allow functions to accept input(s) and perform a task using the input(s). We use parameters as placeholders for information that will be passed to the function when it is called
  • The values that are passed to the function when it is called are called arguments. Arguments can be passed to the function as values or variables.
  • To pass back information from the function call, we use a return statement. To create a return statement, we use the return keyword followed by the value that we wish to return. Like we saw above, if the value is omitted, undefined is returned instead. When a return statement is used in a function body, the execution of the function is stopped and the code that follows it will not be executed.
  • To declare a function expression: Declare a variable to make the variable’s name be the name, or identifier, of your function. Assign as that variable’s value an anonymous function created by using the function keyword followed by a set of parentheses with possible parameters. Then a set of curly braces that contain the function body.
  • Arrow functions remove the need to type out the keyword function every time you need to create a function. Instead, you first include the parameters inside the ( ) and then add an arrow => that points to the function body surrounded in { }.
  • An implicit return is a function body composed of a single-line block does not need curly braces. Without the curly braces, whatever that line evaluates will be automatically returned. The contents of the block should immediately follow the arrow => and the return keyword can be removed.
  • Scope defines where variables can be accessed or referenced. We think about scope in relation to blocks because variables can exist either outside of or within these blocks. In global scope, variables are declared outside of blocks. These variables are called global variables. Because global variables are not bound inside a block, they can be accessed by any code in the program, including code in blocks.
  • When a variable is defined inside a block, it is only accessible to the code within the curly braces {}. We say that variable has block scope because it is only accessible to the lines of code within that block. Variables that are declared with block scope are known as local variables because they are only available to the code that is part of the same block.
  • Since control flow dictates that JavaScript files are executed from top to bottom, order matters.
  • Control flow is the order in which a computer executes code in a script.
